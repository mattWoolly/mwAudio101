# SH-101 Architecture & Signal Flow

## 1. Scope and status

This document is the canonical architecture and signal-flow reference for the
mwAudio101 project, a circuit-accurate, SH-101-inspired software synthesizer.
It captures the block diagram, the front-panel section layout, and the
**FROZEN** IC / reference-designator table reconciled against the service-manual
foldout schematic. Later architecture and backlog phases cite this document by
section number, so section numbers are stable citation addresses.

The Roland SH-101 (introduced 1 Nov 1982, produced 1982–1986) is a
**monophonic, single-VCO analog synthesizer** with a hybrid digital control
section: a CPU handles key assignment, the 100-step sequencer, and the
arpeggiator [^sm][^wiki][^vse]. The core analog signal path is strictly serial.

Every substantive claim below carries a source citation. Disputed and
low-confidence items are surfaced explicitly in
[Section 9](#9-confidence-disputes--honest-labels). Anything requiring bench
measurement (Bode plots, ADSR oscilloscope traces, harmonic spectra) is an
**open validation gap** — the mwAudio101 project has made the decision to take
**no physical-unit measurements**, so such items are not delivered facts.

## 2. Canonical block diagram and signal flow

### 2.1 Serial audio path

The SH-101 service-manual block diagram (dated 1 Nov 1982) labels the
panel/signal sections, left to right, in this exact order:
**MODULATOR | VCO | SOURCE MIXER | VCF | VCA** [^sm][^wiki][^vse]. This is the
documented architecture, not inference.

The audio signal flow is [^sm][^wiki][^vse]:

```text
32-key keyboard / CPU ──► D/A ──► VCO control voltage + Gate
                                       │
   ┌───────────────────────────────────┘
   ▼
VCO (CEM3340, IC13) ── Saw
                    ── Pulse/Square (with PWM)
Sub-oscillator      ── ÷2 / ÷4 square / 25% pulse
Noise generator     ── white noise (TR23)
   │
   ▼
SOURCE MIXER  (independent level sliders: Saw | Pulse/Square | Sub-Osc | Noise)
   │
   ▼
VCF (IR3109, IC14)  4-pole / 24 dB/oct self-oscillating low-pass
   │
   ▼
VCA  (BA662 / BA662A)  ENV / GATE mode
   │
   ▼
Audio output (0 dBm max) + Phones
```

The keyboard and CPU (key assigner, sequencer, arpeggiator) drive the D/A
converter that produces the VCO control voltage and the gate signal [^sm].

### 2.2 Modulation and control sources

Two modulation sources feed the audio path [^sm]:

- The **MODULATOR (LFO)** routes to VCO pitch, pulse width, and VCF cutoff, and
  doubles as the clock generator for the arpeggiator/sequencer
  ([Section 5](#5-modulator-lfo--clock)).
- A single **ADSR envelope** is shared by the VCF and the VCA
  ([Section 6](#6-envelope-adsr-and-vca)).

### 2.3 Voice topology summary

The SH-101 is monophonic with a 32-key F-scale keyboard, a 100-step sequencer,
and an up/down/up-down arpeggiator. The CPU (TMP80C49P microcontroller, IC6)
handles key assignment, the sequencer, and the arpeggiator
[^sm][^wiki][^vse][^reconcile-ics].

## 3. Front-panel sections (panel order)

Front-panel sections appear in panel order as **MODULATOR | VCO | SOURCE
MIXER | VCF | VCA**, plus a **CONTROLLER** strip (volume, portamento, transpose,
bend sensitivity) and the mode / sequencer / arpeggiator controls [^sm].

### 3.1 VCO section

The VCO is a single Curtis **CEM3340** oscillator chip (IC13) [^reconcile-ics].
It produces **saw** and **pulse/square** waves with pulse-width modulation
[^sm][^rcmanual][^wiki]:

- Footage selectable **16' / 8' / 4' / 2'** (4-position octave selector) [^sm].
- Pulse-width modulation range **50% → 0%** duty cycle [^sm].
- Tune **±50 cents** [^sm].
- PWM source selectable: MAN (manual / fixed width), A.ENV (VCA envelope),
  F.ENV (VCF envelope), or LFO [^sm][^rcmanual].

The CEM3340 includes an on-die temperature-compensation block (`TEMPCO GEN`,
pins 1/2) feeding a precision multiplier that scales the incoming CV by a
temperature-dependent on-chip voltage
[reverse-engineered: Electric Druid, layered on the manufacturer block diagram;
theory/inference, medium] [^reconcile-cem][^edcem]. The on-chip compensation
largely eliminates the external tempco resistor that other VCOs require, but it
is **not** a complete cancellation of every thermal term; residual drift is
trimmed externally. Because the CEM3340 carries its own tempco, the VCO
frequency-CV network needs no external thermistor (see
[Section 7.2](#72-thermistor-th1-vcf-cutoff-cv-path)).

### 3.2 Sub-oscillator

The sub-oscillator offers three selectable types, generated by **digital
frequency division of the VCO** [^edsubosc][^rcmanual][^sm]:

- **1 OCT DOWN** (square, ÷2),
- **2 OCT DOWN** (square, ÷4),
- **2 OCT DOWN narrow** (25% pulse, ÷4).

The divider is a dual D-type flip-flop, **MB84013B** (the Matsushita equivalent
of the CD4013), dividing the VCO by 2 and by 4. The two flip-flop outputs are
combined through a diode-OR to create the 25%-duty pulse, which has the
strongest 2nd harmonic [^edsubosc][^sm]. The original sub-osc divider logic ran
at the **+14 V** internal logic rail
[reverse-engineered: AMSynths / Electric Druid, medium] [^amsmod][^edsubosc] —
see [Section 8.2](#82-internal-rails).

### 3.3 Noise generator

The noise source is a **white-noise generator built from a single transistor**,
**TR23 = 2SC945**, factory-selected for noise [^sm][^modwig]. The service-manual
parts list lists 2SC945 (NZ) as the noise generator; the IC/transistor table
notes TR23 is "Selected for Noise" [^sm]. Per AMSynths' reverse-engineering, the
noise feeds a fixed **−3 dB filter at ~16 kHz** before the mixer
[reverse-engineered: AMSynths, medium] [^amsmod].

### 3.4 SOURCE MIXER

The SOURCE MIXER has **independent level controls** for four inputs named on the
Roland panel as **Pulse/Square wave, Saw wave, Sub Osc, and Noise**, each with
its own level slider [^sm][^rcmanual][^wiki]. The original SH-101 has one VCO
with three simultaneously mixable waveforms (Pulse/PWM, Saw, Sub) plus noise.
The exact mixer summing law and per-slider taper (linear vs. audio) are **not**
specified by Roland — see [Section 9](#9-confidence-disputes--honest-labels).

### 3.5 VCF section

The VCF is a Roland **IR3109** (IC14), implementing a **4-pole, 24 dB/octave
low-pass filter that self-oscillates** [^reconcile-ics][^edfilter][^amsfilter].
Details are in [Section 4](#4-vcf-ir3109-detail).

### 3.6 VCA section

The VCA is built around the **BA662 / BA662A** OTA-style gain cell
[^reconcile-vca]. The VCA `MODE` switch chooses **ENV** (volume follows the
ADSR) or **GATE** (fixed volume while a key is held) [^rcmanual][^sm]. The BA662
internals are **reverse-engineered with no public datasheet**
[reverse-engineered: Open Music Labs, no public datasheet]; treat any internal
node behavior accordingly.

### 3.7 MODULATOR section

The MODULATOR (LFO/CLK) section carries the LFO/CLK RATE control, a waveform
selector, HOLD, and EXT CLK IN [^sm][^amsmod]. Detail is in
[Section 5](#5-modulator-lfo--clock).

### 3.8 CONTROLLER strip

The CONTROLLER section provides [^sm][^rcmanual][^rolandsupport]:

- **Volume** (master output level),
- **Portamento Time** (0–5 s) with **Mode** OFF / AUTO (legato only) / ON
  (always),
- **Transpose** (L / M / H keyboard octave shift),
- **VCO Bend Sens** and **VCF Bend Sens**,
- an **LFO MOD** amount tied to the bender lever.

The bender lever applies pitch bend (VCO BEND SENS) and filter bend (VCF BEND
SENS) and, with the LFO MOD switch, applies LFO modulation (sine, derived in the
bender path) to the VCO and/or VCF [^sm][^rcmanual][^amsmod].

## 4. VCF (IR3109) detail

### 4.1 Topology

The IR3109 implements the typical Roland 4-pole, 24 dB/oct low-pass topology in
a configuration similar to the Juno-6/60: **24 dB low-pass with self-oscillation
but no Q compensation**, with resonance feedback limited by **clipping diodes to
ground** rather than on-chip resonance control (which the IR3109 lacks). It will
self-oscillate below 20 Hz and beyond 20 kHz [^edfilter][^amsfilter][^sm]. The
cutoff range is **10 Hz – 20 kHz** with **0–100% key follow**, from the
service-manual spec sheet [^sm].

### 4.2 Integrator / pole stage values (frozen)

Read directly off the high-resolution VCF crop of the foldout schematic, each of
the four IR3109 integrator stages uses **240 pF** (C47, C48, C49, C50) with a
**68 kΩ** input resistor (R114, R117, R118, R121) and a **560 Ω** series
resistor (R115, R116, R120, R123); C45 = 10 µF/16 V, R125 = 820 Ω
[service-manual, high] [^reconcile-vcf][^edfilter]. This matches the Juno-6/60
standard 68 kΩ + 240 pF ladder. An earlier OCR token of "260p" was a misread of
"240p", and the integrator caps are therefore **frozen at 240 pF**, not
single-reader-inferred.

### 4.3 Resonance behavior

The VCF resonance uses a **transistor phase-splitter with clipping diodes to
ground** rather than input-gain compensation: the first transistor stage is
configured as a phase splitter producing inverted/non-inverted copies, with the
inverted signal fed back to the input for resonance, plus clipping diodes to
ground in the feedback path. The SH-101 compensates the resonance volume drop by
**raising output level rather than input level**
[reverse-engineered: Electric Druid / Open Music Labs; some designator-to-
function mappings inferred, medium] [^edfilter][^reconcile-vcf]. The precise
resonance/self-oscillation onset law in SH-101-specific component values was
**not** extracted from the schematic images (only described qualitatively) — see
[Section 9](#9-confidence-disputes--honest-labels).

### 4.4 Clone-derived electrical figures

Any IR3109 electrical figures circulating in the community (e.g. internal drive
currents or a ~20 Vpp self-oscillation amplitude) originate from the **Alfa
AS3109 clone** or **AMSynths module** measurements, **not** from
original-instrument measurements
[clone-derived: Alfa AS3109 / AMSynths, presumed-equal]. They must be labeled as
such wherever used and not treated as original-hardware data.

## 5. MODULATOR (LFO) / clock

### 5.1 LFO core and waveform set

The LFO core generates **square and triangle** at **0.1–30 Hz** from a dual
op-amp [reverse-engineered: AMSynths, medium] [^amsmod][^sm]. The exact op-amp
identity is **unmeasured theory/inference** and is not frozen.

The **original SH-101 panel waveform set is {Square, Triangle, Random, Noise}** —
a 4-way selection [^amsmod]. **SINE is NOT a panel position**: it is derived on
the bender PCB by converting the triangle to a sine, used for the bender/grip
pitch-mod path [^amsmod]. Earlier "(effectively) square/triangle, RANDOM, NOISE"
phrasing is superseded by this cleaner statement.

- **RANDOM** is a CPU-generated 6-bit pseudo-random value written to the DAC each
  clock cycle and amplified ×2 to give a 0…+10 V pseudo sample-and-hold signal
  [reverse-engineered: AMSynths, medium] [^amsmod].
- **NOISE** comes from the analog noise generator [^amsmod].

### 5.2 LFO as clock

The MODULATOR is labeled **LFO/CLK** on the panel; its rate spec is
**LFO/CLK RATE (0.1–30 Hz)**. The **square** waveform is the internal clock for
the arpeggiator and the up-to-100-step sequencer; an **EXT CLK IN** jack allows
external clocking. The CPU resets the clock when a key is pressed (Clock Reset)
[^sm][^amsmod][^wiki].

## 6. Envelope (ADSR) and VCA

### 6.1 Shared single ADSR

There is **one ADSR** (Attack, Decay, Sustain, Release) shared as the
time-varying source for **both** the VCF (ENV amount with selectable
polarity/direction) and the VCA [^rcmanual][^sm]. Service-manual ranges
[^sm]:

- Attack **1.5 ms – 4 s**,
- Decay **2 ms – 10 s**,
- Sustain **0–100%**,
- Release **2 ms – 10 s**.

The ADSR **segment-curve law** (exponential vs. linear shaping) is **unmeasured
theory/inference** and is an open validation gap pending bench data
[theory/inference, unmeasured].

### 6.2 VCA and envelope-trigger modes

- The **VCA MODE** switch chooses **ENV** (volume follows ADSR) or **GATE**
  (fixed volume while a key is held) [^rcmanual][^sm].
- The **ENV TRIG** source selector chooses **GATE+TRIG** (re-triggered on every
  key press), **LFO** (re-triggered each LFO cycle while a key is held), or
  **GATE** (triggered only on a newly pressed key, not on legato)
  [^rcmanual][^sm].

## 7. FROZEN IC / reference-designator table

The following reference-designator assignments are **FROZEN** by schematic
reconciliation against the synthfool foldout scans and ManualsLib pages. They
override any earlier OCR-derived reading. In particular, the round-1 **"IC3"**
reading for the VCO was a **wrong OCR** and is rejected; the VCO is **IC13**
[^reconcile-ics].

### 7.1 Core IC assignments (frozen)

| Designator | Part | Function | Confidence | Source |
| --- | --- | --- | --- | --- |
| IC13 | CEM3340 | VCO | high | synthfool sh101_7 VCO crop + ManualsLib p.9 [^reconcile-ics] |
| IC14 | IR3109 | VCF (4-pole, 24 dB/oct) | high | synthfool sh101_7 VCF crop + ManualsLib p.9 [^reconcile-ics] |
| IC6 | TMP80C49P-6-7301 | CPU (key assign, sequencer, arpeggiator, random) | high | synthfool sh101_3 + ManualsLib p.9/10 [^reconcile-ics] |
| — | BA662 / BA662A | VCA gain cell | high | synthfool sh101_7 + ManualsLib parts list p.10 [^reconcile-vca] |
| — | MB84013B | Sub-osc divider (dual D flip-flop, ÷2/÷4) | high | parts list + Electric Druid [^edsubosc][^sm] |
| TR23 | 2SC945 (NZ) | Noise generator (factory-selected) | high | service-manual parts list [^sm][^modwig] |
| TH1 | SDT-1000 (part 12389800) | Thermistor in VCF cutoff-CV path | high | synthfool sh101_7 + parts list p.10 [^reconcile-th] |

The **"IC3" reading was a wrong OCR**; the archive.org djvu OCR that returned
"IC14 CEM3340" was a known mis-row (IC14 is the IR3109). The chip body printed
**"IC13 CEM3340"** was read directly off the 3000 px scan after region-cropping
and 2× upscaling, and ManualsLib page 9 independently lists "IC13 CEM3340"
[^reconcile-ics].

### 7.2 Thermistor TH1 (VCF cutoff-CV path)

**TH1 (SDT-1000) is in the VCF cutoff-CV path, not the VCO frequency-CV
network** [service-manual, high] [^reconcile-th]. It is wired in series with
R127 (5.6 kΩ) and R126 (1.8 kΩ), adjacent to TR28 and R133 (12 kΩ), in the
op-amp network that scales the VCF cutoff CV feeding the IR3109. The VCO
frequency-CV network around the CEM3340 uses only fixed resistors and trimmers
(VR6 VCO WIDTH, VR7 VCO TUNE) with the +15 V tune reference and **no
thermistor**. This is the canonical Roland split: the CEM3340 is **on-die
temp-compensated** (see [Section 3.1](#31-vco-section)), whereas the OTA-based
IR3109 VCF has no on-die tempco and uses TH1 to stabilize cutoff tracking
[^reconcile-th][^reconcile-cem].

## 8. Power and I/O

### 8.1 I/O and gate logic (frozen)

Service-manual spec sheet I/O [^sm][^reconcile-gate]:

- **CV** scaling **1 V/octave** (0.415 V – 5 V) [^sm].
- **Gate output** OFF = 0 V, **ON = 12 V** per the **service manual**
  [service-manual, high] [^reconcile-gate]. **The service manual is
  authoritative and the project freezes ON = 12 V.** Note: Roland's current web
  spec (official support / Sweetwater) states **ON = 10 V at a 100 kΩ load**;
  this is a genuine documented discrepancy surfaced in
  [Section 9](#9-confidence-disputes--honest-labels). The open-circuit reading
  would plausibly be higher than the loaded reading, explaining 12 vs. 10.
- **Gate / EXT CLK input threshold** +2.5 V or more [^sm].
- **Audio output** 0 dBm max, plus a phones jack [^sm].
- **DC input** 9–12 V (or 6× 1.5 V dry cells); power consumption 1 W; weight
  4.1 kg [^sm][^reconcile-power].

### 8.2 Internal rails

The internal rail set is **+15 V, −15 V and +5 V**, with an additional **+5 V
backup rail** for CPU/sequencer memory, derived from the external 9–12 V DC
input (or 6× 1.5 V dry cells) via a **DC/DC converter** (part 15229908)
[service-manual, medium for rail derivation; DC/DC presence high]
[^reconcile-rails][^reconcile-dcdc]. The +15 V rail feeds the CEM3340
frequency-control network and the VCO-tune reference [^reconcile-rails].

Separately, AMSynths / Electric Druid report a battery-referenced **+14 V**
internal logic rail (with +5 V), powering the MB84013B sub-osc divider
[reverse-engineered: AMSynths / Electric Druid, medium] [^amsmod][^edsubosc].
This **+14 V logic rail is distinct from the gate-output ON level** (12 V
service-manual / 10 V official web spec). Do **not** conflate them.

A third-party mod reference describes additional **+14 V / −5 V sub-rails**
generated by op-amp regulators, but that page was inaccessible to verify and
these sub-rails could **not** be read off the PSU corner of the foldout. They
are a **labeled dispute**, not frozen — see
[Section 9](#9-confidence-disputes--honest-labels).

## 9. Confidence, disputes & honest labels

This section surfaces every disputed, low-confidence, or honestly-labeled item
for this dimension. None of the following are settled facts.

### 9.1 Disputed and uncertain items

- **Gate-output ON level (DISPUTED).** Service manual says **12 V**; official
  Roland support and Sweetwater say **10 V at 100 kΩ load** [^reconcile-gate]
  [^rolandsupport]. The OFF = 0 V part is unanimous. The project **freezes
  ON = 12 V (service manual authoritative)** but records the conflict; do not
  assert a single value at high confidence in downstream prose. Likely an
  open-circuit vs. loaded-measurement difference.
- **Extra +14 V / −5 V PSU sub-rails (DISPUTED, low).** Reported by an
  inaccessible third-party (synhouse.com) and **not** read off the schematic;
  the exact regulator topology of the PSU corner did not resolve to legible text
  [theory/inference, unmeasured] [^reconcile-railsdisp]. Treat the rail set
  beyond {+15 V, −15 V, +5 V (+ 5 V backup)} as low confidence.
- **MGS-1 / MG-1 grip electrical interface (DISPUTED, low).** Confirmed it is a
  purely **analog** interface (pitch bend + LFO-mod switch via dedicated side
  jacks) — the SH-101 is **pre-MIDI, so the grip sends NO MIDI CC at all**
  [^reconcile-grip][^mgs1][^kvr]. The set is sold as **MGS-1**; the grip unit is
  labeled **MG-1** in the owner's manual. The exact connector pinout and per-pin
  signal assignment are **undocumented** in reviewed sources; do not assume an
  interface in the circuit model.

### 9.2 Reverse-engineered / clone-derived / inferred items

- **CEM3340 on-die tempco completeness** — manufacturer block diagram is solid,
  but the "not a complete cancellation of every thermal term" commentary is
  [theory/inference, medium; reverse-engineered: Electric Druid]
  [^reconcile-cem][^edcem].
- **VCF resonance designator-to-function mapping** — phase-splitter + clipping
  diodes confirmed in concept, but some designator mappings are
  [reverse-engineered: Electric Druid / Open Music Labs, partly inferred]
  [^edfilter][^reconcile-vcf].
- **IR3109 electrical figures** (drive currents, ~20 Vpp self-osc) are
  [clone-derived: Alfa AS3109 / AMSynths, presumed-equal] — **not**
  original-instrument measurements [^amsfilter][^edfilter].
- **BA662 / BA662A VCA internals** are [reverse-engineered: Open Music Labs, no
  public datasheet] [^reconcile-vca].
- **LFO core op-amp identity** is [theory/inference, unmeasured]; the dual-op-amp
  square+triangle core itself is [reverse-engineered: AMSynths, medium]
  [^amsmod].
- **ADSR segment-curve law** is [theory/inference, unmeasured]; the time ranges
  are [service-manual, high] [^sm].
- **Noise −3 dB @ ~16 kHz filter** and **6-bit random S&H** mechanism come from a
  single high-quality secondary source [reverse-engineered: AMSynths, medium]
  [^amsmod]; not node-by-node cross-checked against the raw schematic.
- **+14 V logic rail** is well-sourced for the sub-osc Vdd but is a teardown-
  derived value, not a Roland spec-sheet figure [reverse-engineered:
  AMSynths / Electric Druid, medium] [^amsmod][^edsubosc].

### 9.3 Open validation gaps (no physical-unit measurements)

The mwAudio101 project takes **no bench measurements** by decision. The following
are open validation gaps, not delivered facts:

- VCF Bode/transfer-function plots and the exact resonance self-oscillation onset
  in SH-101-specific values [^reconcile-vcf].
- ADSR oscilloscope segment curves [^sm].
- Harmonic spectra of saw/pulse/sub waveforms.
- Mixer summing law and per-slider taper (linear vs. audio) — Roland gives no
  slider scaling [^sm].
- VCF ENV-amount polarity/bipolarity numeric range — implied but not confirmed
  numerically [^rcmanual].
- Whether the bender-derived sine and the panel LFO share one oscillator (with a
  waveshaper) or two paths — sources agree the triangle is converted to sine on
  the bender PCB, but the shared-vs-separate routing at the LFO output stage is
  not fully pinned down [^amsmod].

### 9.4 Software-emulation artifacts (NOT original hardware)

The following appear only in **later software** (e.g. Roland Cloud SH-01A) and
are **NOT** in the original hardware; do not model them as original-instrument
features:

- **No sine LFO** as a panel waveform (sine exists only in the bender path)
  [^amsmod].
- **No 32' / 64' registers** (footage is 16'/8'/4'/2' only) [^sm].
- **No external audio input.**
- **No MIDI / DCB** — the SH-101 is pre-MIDI; control is CV/Gate only [^reconcile-grip].

### 9.5 OCR caveat

Several panel/spec values (footage, ADSR, cutoff, CV, thresholds, audio level)
rest on archive.org OCR of the service manual, which can mis-read characters
(the range glyph, and the very 10-vs-12 ambiguity at issue). Values were
sanity-checked against secondary sources and the high-resolution synthfool/
ManualsLib scans and are internally consistent, but a clean primary scan should
be the citation of record [^sm][^reconcile-gate].

## 11. Design implications for mwAudio101

For the mwAudio101 DSP model, implement a **strictly serial mono voice** that
mirrors the canonical path of [Section 2](#2-canonical-block-diagram-and-signal-flow)
[^sm]:

1. **Oscillator** — one band-limited oscillator producing saw + variable-width
   pulse with PWM 50% → 0% ([Section 3.1](#31-vco-section)).
2. **Sub-oscillator** — a digitally-divided sub (÷2 square, ÷4 square, and a
   diode-OR'd 25% pulse at ÷4) plus a white-noise generator
   ([Sections 3.2–3.3](#32-sub-oscillator)).
3. **Source mixer** — a 4-input mixer (Saw, Pulse/Square, Sub-Osc, Noise) with
   independent gains. The summing law/taper is an open gap
   ([Section 9.3](#93-open-validation-gaps-no-physical-unit-measurements)); start
   with independent linear gains and revisit.
4. **VCF** — a 24 dB/oct (4-pole) self-oscillating low-pass modeled on the
   IR3109: resonance to self-oscillation with **diode-clipped feedback** and
   **no Q makeup gain** (level drops as resonance rises)
   ([Section 4](#4-vcf-ir3109-detail)). Use the frozen 68 kΩ + 240 pF + 560 Ω
   ladder values as the integrator-stage reference
   ([Section 4.2](#42-integrator--pole-stage-values-frozen)).
5. **VCA** — ENV / GATE modes ([Section 3.6](#36-vca-section)).
6. **Envelope** — a single ADSR (A 1.5 ms–4 s; D/R 2 ms–10 s; S 0–100%) shared
   by VCF (bipolar amount + polarity) and VCA. The curve law is an open gap;
   choose a documented default and flag it ([Section 6](#6-envelope-adsr-and-vca)).
7. **LFO / clock** — one LFO/CLK (0.1–30 Hz) whose primitive shapes are
   **triangle and square**, with **sine generated by shaping the triangle for
   the bender path only** (not a panel position), plus CPU-style sample-and-hold
   "random" and band-limited noise. Route the LFO to VCO pitch, PWM, and VCF
   cutoff, and reuse the **square as the arpeggiator/sequencer clock** with
   key-press clock-reset ([Section 5](#5-modulator-lfo--clock)).
8. **Controller params** — portamento (0–5 s; OFF/AUTO/ON), transpose L/M/H,
   VCO/VCF bend sensitivity, master volume ([Section 3.8](#38-controller-strip)).
9. **Calibration / I/O** — calibrate pitch to **1 V/oct**; if emulating gate
   logic, use the project-frozen **+12 V** output level
   ([Section 8.1](#81-io-and-gate-logic-frozen)), and keep the 10 V web-spec
   conflict noted.

UI/panel layout should mirror the canonical left-to-right section order
**MODULATOR | VCO | SOURCE MIXER | VCF | VCA** with the controller strip and
100-step sequencer/arpeggiator, using the exact original legends (FEET, PULSE
WIDTH, MOD, SUB OSC TYPE, FREQ/RES/ENV/MOD/KYBD, ADSR, VCA MODE, ENV TRIG,
PORTAMENTO, TRANSPOSE) [^sm][^rcmanual].

**Do not** add software-only artifacts (sine LFO panel position, 32'/64'
registers, external audio input, MIDI/DCB) to the original-hardware model — see
[Section 9.4](#94-software-emulation-artifacts-not-original-hardware).
